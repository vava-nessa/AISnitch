# Daemon-supervised fullscreen dashboard

## Context

Make the fullscreen web dashboard available whenever the AISnitch daemon is running, instead of only while `aisnitch fs` keeps a foreground helper alive. `aisnitch start` should start the daemon by default, expose the web dashboard on `127.0.0.1:5174`, and show that URL in the TUI. `aisnitch fs` should ensure the daemon/dashboard are ready, open the browser, then return cleanly.

## Subtasks

- [x] Add dashboard port configuration
  report: Added `dashboardPort` to the runtime config schema with default port 5174.
- [x] Start the dashboard server from daemon mode
  report: Daemon (`runPipelineHeadless` with daemonMode=true) spawns the dashboard server via `startDashboardServerProcess` using packaged static assets and Node built-ins only.
- [x] Make `aisnitch` (no command) the default that opens the dashboard in the browser
  report: Moved `isDefault: true` from `start` to `fs`. `aisnitch start` keeps the TUI, while `aisnitch` alone starts the daemon, ensures the dashboard server is up, and opens the browser.
- [x] Make `aisnitch fs` open the daemon-managed dashboard with self-healing
  report: `fullscreen()` now starts the daemon if needed, falls back to spawning a standalone dashboard server when an existing daemon is up but not serving the dashboard, and surfaces browser-open errors with a manual fallback URL.
- [x] Show the dashboard URL in CLI status and the TUI
  report: `Header.tsx` shows `Web <dashboardUrl>`, `StatusBar.tsx` shows `Daemon active ¬∑ Web <dashboardUrl>`, and `aisnitch status` includes the dashboard URL.
- [x] Update runtime tests for the new lifecycle
  report: Adjusted the failure-surfacing test to expect `start()` itself to throw, and removed the toggleDaemon call from the info-log test to avoid `process.kill(process.pid)`.
- [x] Update documentation and task notes
  report: README quick-start now leads with `aisnitch` instead of `aisnitch start`, and the Web Dashboard section advertises the no-arg form. CLI help examples list `aisnitch` as the default.
- [x] Run lint, typecheck, tests, build, and package dry-run
  report: `pnpm lint`, `pnpm typecheck`, `pnpm test` (333 tests passing), and `pnpm build` all green.

## Notes

- The dashboard server must use packaged static assets and Node built-ins only, so global npm/Homebrew installs do not require Vite at runtime.
- The current published version before this task is `0.2.25`; target patch release is `0.2.26`.
